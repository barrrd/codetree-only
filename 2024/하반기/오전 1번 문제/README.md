# [삼성 기출] 미지의 공간 탈출

> **문제 링크**: [코드트리 - 미지의 공간 탈출]()

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 람다(Lambda)를 이용한 면 사이의 좌표 및 축 변환
정육면체의 윗면과 옆면 사이를 이동할 때, 면의 회전 상태에 따라 행(`r`)과 열(`c`)이 서로 바뀌거나 인덱스가 반전(`m-1-x`)되는 구간이 발생합니다.

* **오류 예시**: 단순히 `nr=0` 또는 `nr=m-1`로만 고정하여 이동시키면, 전개도상에서 90도 회전하여 맞닿은 모서리의 인덱스 규칙을 위반하게 됨.
* **핵심**: `side_move` 딕셔너리에 `(현재면, 방향)`을 키로 하여 **다음 좌표를 계산하는 함수**를 사전 정의해야 합니다.
* **➡️ 해결 방법**:
    * `side_move = {(cp, d): (lambda r, c: ..., lambda r, c: ..., next_p)}` 구조 활용.
    * **대칭성 검증 필수**: 4번(위)에서 0번(동)으로 갔을 때의 좌표 변환과, 0번에서 4번으로 돌아올 때의 좌표 변환이 수학적으로 역관계인지 확인 (예: `r`이 `c`로 변했다면 돌아올 땐 `c`가 `r`로 변해야 함).

### 2) 인덱스 범위 체크의 이원화 ($M$ vs $N$)
정육면체 면의 크기($M$)와 바닥 평면도의 크기($N$)가 다르므로, 현재 위치에 따라 참조할 배열과 경계값을 엄격히 분리해야 합니다.

* **오류 예시**: 정육면체 위에서 이동 중인데 전체 맵 크기인 `n`을 기준으로 `if nr < n` 조건을 걸어버리면, `phase[side][m][m]` 배열 참조 시 `IndexError` 발생.
* **핵심**: `cp < 5`(정육면체)일 때는 `m`을 기준으로, 바닥으로 내려온 뒤에는 `n`을 기준으로 경계값을 체크합니다.
* **➡️ 해결 방법**:
    * `phase` 배열을 `[5][M][M]` 크기로 선언하여 정육면체 전용 방문 체크 수행.
    * `if not (0 <= nr < m and 0 <= nc < m):` 로직을 통해 정육면체 모서리를 넘었을 때만 면 전환 로직(`side_move`) 실행.

### 3) 시간 이상 현상(확산)의 시뮬레이션 순서
이상 현상은 매 $v_i$ 배수 턴마다 특정 방향으로 증식하며, 기존 위치도 유지됩니다.

* **오류 예시**: 플레이어가 먼저 이동하고 나중에 확산시키면, 플레이어가 도착한 칸에 동시에 확산되는 장애물을 피하지 못하는 판정 오류 발생.
* **핵심**: **[이상 현상 확산 ➡️ 플레이어 BFS 이동]** 순으로 한 턴을 구성하거나, 이상 현상이 각 칸에 도달하는 시간을 미리 계산해 두어야 합니다.
* **➡️ 해결 방법**:
    * `if t // v > 0 and t % v == 0:` 조건을 사용하여 확산 지점 업데이트.
    * 확산은 "빈 공간"으로만 퍼지므로, `arr_2d`의 장애물과 탈출구 정보를 반드시 참조하여 `continue` 처리.

### 4) BFS 탐색 시 변수 오염 방지 (`cp` vs `np`)
한 위치에서 사방(동, 서, 남, 북)을 탐색할 때, 한 방향에서 면이 바뀌었다고 해서 기준 면이 변하면 안 됩니다.

* **오류 예시**: `for` 루프 내부에서 현재 면 번호인 `np`를 직접 수정해버리면, 다음 방향 탐색 시 원래 면이 아닌 바뀐 면을 기준으로 좌표를 계산하게 됨.
* **핵심**: 큐에서 꺼낸 현재 면 번호(`cp`)는 고정하고, 이동할 다음 면 번호(`np`)는 `for` 루프 내부에서 매번 초기화하여 사용해야 합니다.
* **➡️ 해결 방법**:
    ```python
    while q:
        cr, cc, cp, turn = q.popleft()
        if (cr,cc,cp) == (tr,tc,tp):
            answer = turn
            break

        for i in range(4):
            nr, nc = cr + dr[i], cc + dc[i]
            nxt_turn, np = turn + 1, cp
            # 격자 밖
            if not(0 <= nr < m and 0 <= nc < m):
                if np in [0,1,2,3] and i == 2:
                    continue
                func_r, func_c, np = side_move[(np,i)]
                nr, nc = func_r(cr,cc), func_c(cr,cc)
            if phase[np][nr][nc] <= nxt_turn or arr_3d[np][nr][nc] == 1: # 이미 방문 과 벽
                continue
            phase[np][nr][nc] = nxt_turn
            q.append((nr,nc,np,nxt_turn))
    ```

### 5) 옆면 하단에서 바닥으로의 탈출 판정
옆면(0~3)에서 남쪽(`d=2`)으로 이동하는 것은 면 전환이 아니라 바닥 평면도로의 '추락'입니다.

* **오류 예시**: `side_move`에 남쪽 이동을 정의하지 않아 `KeyError`가 발생하거나, 바닥 좌표 계산을 누락하는 경우.
* **핵심**: 옆면의 아래쪽 경계에 닿았을 때는 바닥의 시작 좌표 `(R, C)`와 옆면의 상대 좌표를 조합하여 바닥의 절대 좌표를 도출해야 합니다.
* **➡️ 해결 방법**:
    * `if cp in [0, 1, 2, 3] and i == 2:` 예외 처리를 통해 바닥 진입 로직 분리.
    * 바닥 진입 시점의 `turn`과 이상 현상 발생 시간을 비교하여 생존 여부 판정.
