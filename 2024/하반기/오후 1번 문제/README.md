# [삼성 기출] 메두사와 전사

> **문제 링크**: [코드트리 - 메두사와 전사](https://www.codetree.ai/ko/frequent-problems/samsung-sw/problems/medusa-and-warriors/description)

---

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 메두사의 기본 시야각(90도) 영역 설정
메두사의 시선 처리는 그림자를 계산하기 전에, 먼저 **메두사가 바라보는 90도 쐐기 모양의 기본 시야 범위(FOV)**를 정확히 잡아내는 것부터 시작해야 합니다.

* **오류 예시:** 시야각을 단순히 반복문의 인덱스 증감만으로 구현하려고 하면 범위를 벗어나거나 코드가 지나치게 복잡해집니다.
* **핵심:** 메두사의 좌표 `(mr, mc)`와 탐색 칸 `(r, c)`의 **행렬 좌표 차이의 절댓값**을 이용하면 90도 영역을 아주 수학적이고 깔끔하게 정의할 수 있습니다.
* **➡️ 해결 방법:**
    * 상/하(`d=0, 1`)를 볼 때는 가로 거리보다 세로 거리가 길거나 같아야 함: `abs(c - mc) <= abs(r - mr)`
    * 좌/우(`d=2, 3`)를 볼 때는 세로 거리보다 가로 거리가 길거나 같아야 함: `abs(r - mr) <= abs(c - mc)`
    * 이 조건에 맞는 칸들만 `view_map = True`로 설정하고, 해당 영역 안의 전사들을 `targets` 배열에 담아 메두사와의 거리순으로 정렬합니다.

### 2) 시야 내 전사들에 의한 그림자 전파 (45도 대각선 vs 직선)
기본 시야각을 잡은 후, 가까운 전사부터 뒤편으로 그림자를 드리워 `view_map`을 `False`로 지워나가야 합니다. 이때 전사의 상대적 위치(정면 vs 대각선)에 따라 그림자 형태가 완전히 달라집니다.

* **오류 예시:** 대각선 방향의 전사가 가리는 그림자를 90도 꽉 찬 직사각형으로 덮거나, 모든 그림자를 삼각형으로 통일할 경우 가려지지 않아야 할 전사가 돌이 되는 현상이 발생합니다.
* **핵심:** 전사가 메두사의 **정면(같은 행/열)**에 위치할 때는 그림자가 **일직선**으로만 생깁니다. 반면, **대각선 영역**에 있을 때는 전사를 기준으로 **45도 쐐기(삼각형)** 형태로 그림자가 퍼져나가야 합니다.
* **➡️ 해결 방법:**
    * 방향(`d`) 분기 후, 전사와 메두사의 상대 좌표 차이(`rc = tc - mc` 또는 `rr = tr - mr`)를 확인합니다.
    * `rc == 0` (정면)일 때는 `view_map[r][tc] = False`와 같이 해당 열/행만 직선으로 지웁니다.
    * `rc != 0` (대각선)일 때는 `abs(c - tc) <= abs(r - tr)` 구조의 수식을 활용하여 45도 각도로 퍼지는 그림자 범위만 `False`로 지워나갑니다.

### 3) 전사의 1차, 2차 이동 방향 우선순위 분리
전사들은 최대 2칸을 이동하는데, 첫 번째 이동과 두 번째 이동의 방향 우선순위가 지문에 다르게 명시되어 있습니다.

* **오류 예시:** 1차 이동과 2차 이동을 같은 방향 우선순위 배열을 쓰거나, 모듈러 연산(`%4`) 등으로 인덱스를 억지로 끼워 맞출 경우 오작동이 발생합니다.
* **핵심:** 1차 이동은 **[상 ➡️ 하 ➡️ 좌 ➡️ 우]**, 2차 이동은 **[좌 ➡️ 우 ➡️ 상 ➡️ 하]**의 우선순위를 엄격하게 지켜야 합니다.
* **➡️ 해결 방법:**
    * `for mul in range(2):` 루프 내부에서 이동 차수에 따라 명시적인 리스트를 선언합니다.
    * `mul == 0` (1차 이동): `dir_order = [0, 1, 2, 3]`
    * `mul == 1` (2차 이동): `dir_order = [2, 3, 0, 1]`
    * 위 리스트의 순서대로 `dr, dc`를 참조하여 거리가 줄어드는지(`nxt_dist < curr_dist`) 확인합니다.

### 4) 전사의 타격 및 소멸 시 누적 거리 계산
전사가 1~2칸 이동하다가 메두사와 같은 칸에 도달하면 메두사를 타격하고 즉시 소멸합니다. 이때 이동 거리에 대한 처리를 주의해야 합니다.

* **오류 예시:** 메두사 칸에 도달하여 소멸(`loop1 = True`)했다고 해서, 해당 턴에 그 전사가 이동한 거리를 전체 이동 거리 합산(`ans0`)에서 누락시키는 실수.
* **핵심:** 메두사를 만나 소멸하더라도 죽기 전까지 뛰어간 거리는 무조건 총합에 더해져야 하며, 생존 여부에 따라 다음 턴 격자(`nxt_warr`) 기록 여부만 갈려야 합니다.
* **➡️ 해결 방법:**
    * 2칸 이동 탐색 루프가 끝난 직후, 생존 플래그(`loop1`)와 무관하게 `ans0 += total_move * warr[r][c]`를 먼저 처리합니다.
    * 그 후 `if not loop1:` 일 때만 `nxt_warr[nxtr][nxtc] += warr[r][c]`를 수행하여 살아남은 전사만 맵에 남깁니다.
