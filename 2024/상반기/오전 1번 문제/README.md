# 🏺 유물 탐사(Artifact Exploration) 구현 회고

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 격자 회전 시 인덱스 범위와 "중심점"의 착각
3x3 격자를 회전시킬 때, 선택한 중심 좌표 $(r, c)$를 기준으로 실제 배열 인덱스를 계산하는 과정에서 실수가 잦습니다.
- **오류 예시:** 5x5 배열에서 3x3 격자를 돌릴 때 시작점을 `range(1, 4)`로 설정하면, 인덱스가 3일 때 `3, 4, 5`를 참조하게 되어 `IndexError`가 발생합니다.
- **핵심:** 3x3 격자의 왼쪽 상단(시작점)을 기준으로 잡을지, 실제 문제에서 말하는 중심점(Center)을 기준으로 잡을지 명확히 정의해야 합니다.
- **➡️ 해결 방법:**
    - 시작점(Top-Left)을 기준으로 `range(0, 3)`을 순회하며 `cw(arr, r, c)` 함수에 전달합니다.
    - 함수 내부에서는 `arr[r][c]`부터 `arr[r+2][c+2]`까지의 3x3 영역만 정확히 회전시키도록 범위를 제한합니다.



### 2) 다중 조건 우선순위 정렬 (Python 튜플의 특성 활용)
이 문제의 가장 큰 난관은 **1. 유물 가치 최대, 2. 각도 최소, 3. 열 최소, 4. 행 최소**라는 4단계 우선순위를 한 번에 처리하는 것입니다.
- **오류 예시:** `if best < temp:` 조건문으로 그때그때 갱신하려 하면, 점수가 같을 때 각도나 좌표 조건을 일일이 `if-elif`로 비교해야 해서 코드가 비대해지고 버그가 생기기 쉽습니다.
- **핵심:** 파이썬의 `sort()`가 튜플의 왼쪽 요소부터 차례대로 비교한다는 특성을 이용합니다.
- **➡️ 해결 방법:**
    - 탐색 시 발생하는 모든 경우의 수(27가지)를 `results.append((-score, angle, col, row, matrix))` 형태로 리스트에 담습니다.
    - 점수는 최대여야 하므로 **마이너스(`-`)**를 붙여 내림차순 효과를 내고, 나머지는 오름차순으로 정렬되도록 구성합니다.
    - `results.sort()` 후 `[0]`번째 요소를 선택하면 복잡한 조건 비교 없이 단번에 최적의 해를 뽑을 수 있습니다.

### 3) 연쇄 폭발(Chain Reaction) 시 "점수 누적"과 "재채우기"의 순서
유물이 터진 후 빈칸이 생기면 새로운 조각이 채워지고, 이로 인해 또다시 유물이 터지는 연쇄 반응이 일어납니다.
- **오류 예시:** 첫 번째 탐사에서 얻은 점수만 더하고 끝내거나, 유물을 지우자마자 채우지 않고 다음 BFS를 돌리면 새로 들어온 조각에 의한 연쇄 반응을 감지할 수 없습니다.
- **핵심:** `while True` 루프를 활용하여 '더 이상 터질 유물이 없을 때까지' **[삭제 ➡️ 채우기 ➡️ 재탐색]** 과정을 반복합니다.
- **➡️ 해결 방법:**
    - `delete_bfs` 함수가 터뜨린 유물 개수를 반환하도록 설계합니다.
    - `while` 내부에서 `count = delete_bfs(arr)`를 호출하고, `count == 0`이면 루프를 탈출합니다.
    - 유물이 터졌을 때만 **열 번호 작은 순 ➡️ 행 번호 큰 순(바닥부터)** 빈칸을 즉시 채워준 뒤 다시 루프를 돌려 연쇄 반응을 확인합니다.



### 4) 빈칸 채우기(Filling) 시의 엄격한 우선순위
조각이 보충되는 순서는 문제에서 제시한 특정 우선순위를 반드시 지켜야 합니다.
- **핵심 규칙:** **(1) 열 번호가 작은 순**, **(2) 행 번호가 큰 순(아래쪽)**
- **➡️ 해결 방법:**
    - `for c in range(5):` (열 우선) 루프 안에 `for r in range(4, -1, -1):` (행은 바닥부터) 루프를 구성합니다.
    - `arr[r][c] == 0`인 곳을 발견하면 `walls.popleft()`를 사용하여 벽면의 조각을 순서대로 채웁니다. 이때 `walls`는 **`collections.deque`**를 사용해야 효율적인 $O(1)$ 연산이 가능합니다.
