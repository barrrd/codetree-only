> **문제 링크**: [코드트리 - 술래잡기](https://www.codetree.ai/ko/frequent-problems/samsung-sw/problems/hide-and-seek/description)

# ⚠️ 구현하면서 헷갈렸던 핵심 포인트

## 술래의 방향 전환 (달팽이 이동)

이 문제의 핵심 난이도는 **술래의 달팽이 이동 방향 전환 로직**이다.

달팽이 패턴은 다음과 같다:

```
1, 1, 2, 2, 3, 3, ..., n-1, n-1, n-1
```

* 마지막 `n-1`은 **3번 등장**해야 (0,0)에 정확히 도달한다.
* 이후에는 반대로

```
n-1, n-1, n-1, n-2, n-2, ..., 1, 1
```

중앙으로 복귀한다.

---

### ✔️ 방향 전환을 위한 핵심 상태 변수

```python
run_len   # 현재 직선에서 이동해야 할 길이
run_cnt   # 현재 직선에서 몇 칸 이동했는지
turn_cnt  # 같은 길이를 몇 번 사용했는지 (2번이면 길이 변경)
```

### ✔️ 기본 로직

```python
run_cnt += 1

if run_cnt == run_len:
    방향 전환
    turn_cnt += 1
    run_cnt = 0

if turn_cnt == 2:
    run_len += 1   # 시계 방향(cw)
    run_len -= 1   # 반시계 방향(ccw)
    turn_cnt = 0
```

---

### ✔️ (0,0) ↔ 중앙 도달 시 처리

이 부분을 정확히 처리하지 않으면 달팽이가 꼬인다.

* (0,0) 도달 → 반시계(ccw) 시작
* 중앙 도달 → 시계(cw) 시작

그리고 반드시:

```python
방향 180도 반전
run_len / run_cnt / turn_cnt 초기화
```

이 전환 타이밍이 정확해야 전체 시뮬레이션이 안정적으로 동작한다.
