# [HSAT 기출] 디지털 로직 패턴 검사

> **문제 링크**: [코드트리 - 디지털 로직 패턴 검사](https://www.codetree.ai/ko/frequent-problems/hsat/problems/check-digital-logic-pattern/description)

---

## ⚠️ 해결 시 주의해야 할 핵심 포인트

### 1) 핵심 아이디어: 길이 `K`만 보면 충분함

문제는 **길이 `K` 이상의 동일한 패턴이 `M`번 이상 반복되는지**를 판단하는 것입니다.

처음 보면 길이 `K`, `K+1`, `K+2`, ... 를 모두 확인해야 할 것처럼 보이지만, 실제로는 **길이 정확히 `K`인 패턴만 검사하면 충분**합니다.

* **이유:**
  어떤 길이 `L (L >= K)`의 동일한 패턴이 `M`번 등장했다면, 그 패턴의 **앞 `K`글자**도 같은 시작 위치에서 똑같이 `M`번 등장합니다.
* **예시:**
  `10110` 이 `M`번 반복되었다면, `101` 역시 `M`번 반복됩니다.

즉 문제는 아래와 같이 바뀝니다.

> **길이 `K`인 부분 문자열 중, `M`번 이상 등장하는 것이 하나라도 있는가?**

---

### 2) 단순 슬라이싱 + 딕셔너리는 시간초과 위험

가장 직관적인 방법은 길이 `K`짜리 부분 문자열을 모두 잘라서 딕셔너리로 세는 것입니다.

```python
for i in range(len(s) - K + 1):
    pattern = s[i:i+K]
```

하지만 이 방식은 매번 길이 `K`짜리 문자열을 새로 생성하므로,
문자열 길이가 최대 `1,000,000`일 때 파이썬에서는 **시간초과**가 날 가능성이 큽니다.

* **오류 원인:**
  슬라이싱 `s[i:i+K]` 자체가 `O(K)` 비용
* **전체 복잡도:**
  대략 `O(N*K)`

---

### 3) Step 1: 비트마스크 + 슬라이딩 윈도우로 최적화

이 문제의 문자열은 오직 `'0'`, `'1'`로만 이루어져 있습니다.
즉 길이 `K`짜리 패턴을 문자열이 아니라 **정수(비트마스크)** 로 표현할 수 있습니다.

예를 들어 `K = 4`일 때:

* `0101` → 정수 `5`
* `1110` → 정수 `14`

이렇게 표현하면 매번 문자열을 잘라 만들 필요 없이,
이전 상태에서 **한 칸 밀고 새 비트를 추가**하는 방식으로 갱신할 수 있습니다.

---

### 4) Step 2: 비트마스크 갱신 방식 이해하기

현재 길이 `K`의 패턴을 `mask`라고 할 때, 새 비트 `bit`를 붙여 다음 윈도우를 만드는 방법은 다음과 같습니다.

```python
mask = ((mask << 1) | bit) & full_mask
```

여기서:

* `mask << 1` : 기존 비트들을 왼쪽으로 한 칸 이동
* `| bit` : 새 비트를 맨 뒤에 추가
* `& full_mask` : 길이 `K`를 초과한 앞부분 제거

예를 들어 `K = 3`, `full_mask = 111(2)` 일 때:

* 현재 `mask = 101`
* 새 비트 `1` 추가
* `101 -> 1011`
* 마지막 3비트만 남기면 `011`

즉 다음 길이 3짜리 패턴으로 자연스럽게 넘어갑니다.

---

### 5) 왜 처음 `mask = 0`으로 시작해도 되는가?

처음에는 `mask = 0`으로 시작하므로, 앞에 실제로 없는 0이 붙는 것처럼 보일 수 있습니다.
하지만 **길이 `K`개를 모두 읽기 전까지는 딕셔너리에 넣지 않기 때문에 문제 없습니다.**

즉 다음 조건이 핵심입니다.

```python
if i >= K - 1:
```

이 조건을 만족하는 순간부터만 길이 `K`짜리 패턴이 완성된 것이므로 카운트합니다.

* **초반 중간 상태:** 카운트 안 함
* **길이 `K` 완성 후:** 실제 패턴으로 카운트

---

### 6) Step 3: 딕셔너리로 등장 횟수 세기

비트마스크로 표현된 각 길이 `K` 패턴의 등장 횟수를 딕셔너리로 관리합니다.

```python
count[mask] = count.get(mask, 0) + 1
```

그리고 어떤 패턴이든 개수가 `M` 이상이 되는 순간 바로 답이 결정됩니다.

* `count[mask] >= M` 이면 → 불안정한 디지털 로직
* 끝까지 없으면 → 안정적인 디지털 로직

---

### 7) 조기 종료(Early Exit) 중요

이미 어떤 패턴의 개수가 `M` 이상이 된 순간, 더 이상 뒤를 볼 필요가 없습니다.

즉:

```python
if count[mask] >= M:
    print(1)
    break
```

처럼 **즉시 종료**하는 것이 효율적입니다.

---

### 8) 구현 시 자주 하는 실수

#### (1) `K` 이상 길이를 전부 검사하려는 실수

이 문제는 길이 `K`만 세면 충분합니다.
길이 `L >= K` 패턴이 반복되면 그 접두 길이 `K` 패턴도 반드시 반복되기 때문입니다.

#### (2) 문자열 슬라이싱으로 시간초과

```python
pattern = s[i:i+K]
```

이 방식은 논리는 맞지만 큰 입력에서 느릴 수 있습니다.
이 문제는 `0/1` 문자열이므로 **비트마스크 방식이 훨씬 유리**합니다.

#### (3) `flag` 초기값 반대로 두기

보통은

* 처음엔 못 찾았으므로 `False`
* 찾으면 `True`

가 더 직관적입니다.

하지만 조기 종료를 쓰면 `flag` 없이도 충분합니다.

#### (4) `K`개가 완성되기 전에 카운트하는 실수

초반에는 길이 `K`짜리 패턴이 아직 만들어지지 않았으므로:

```python
if i >= K - 1:
```

조건을 반드시 걸어야 합니다.

## ✅ 권장 풀이 구조

1. 길이 `K`짜리 패턴만 보면 충분하다는 점을 이용
2. 문자열을 비트마스크 정수로 변환하며 슬라이딩 윈도우 수행
3. 각 패턴의 등장 횟수를 딕셔너리로 관리
4. 어떤 패턴이든 `M`번 이상 나오면 즉시 `1` 출력
5. 끝까지 없으면 `0` 출력

---

## ✅ 핵심 코드 아이디어

```python
digital_logic = input().strip()
K, M = map(int, input().split())

count = {}
mask = 0
full_mask = (1 << K) - 1

for i, ch in enumerate(digital_logic):
    bit = int(ch)
    mask = ((mask << 1) | bit) & full_mask

    if i >= K - 1:
        count[mask] = count.get(mask, 0) + 1
        if count[mask] >= M:
            print(1)
            break
else:
    print(0)
```

---

## ✅ 시간 복잡도

* 각 문자를 한 번씩만 처리
* 딕셔너리 삽입/조회는 평균 `O(1)`

따라서 전체 시간 복잡도는 `O(N)` 수준으로 처리할 수 있습니다.

---

## ✅ 한 줄 핵심 정리

이 문제는 **길이 `K` 이상의 반복 패턴 존재 여부**를 묻지만,
실제로는 **길이 `K`인 패턴의 등장 횟수만 세면 충분**하며,
입력이 `0/1` 문자열이므로 **비트마스크 + 슬라이딩 윈도우 + 딕셔너리**로 빠르게 해결할 수 있습니다.
