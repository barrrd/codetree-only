## ⚠️ 해결 시 주의해야 할 핵심 포인트

### 1) 핵심 아이디어: 파라메트릭 서치 (Parametric Search)
이 문제는 단순히 "길이 `K` 이상의 등산로가 존재하는가?"를 묻는 문제가 아닙니다.  
핵심은 **등산로를 따라 이동할 때 발생하는 인접 높이 차들 중 최댓값을 최소화**하는 것입니다.

즉, 정답을 직접 구하기보다 다음과 같은 **결정 문제**로 바꾸어야 합니다.

* **질문 변형:**  
  **"인접한 두 칸의 높이 차가 모두 `L` 이하가 되도록 하면서, 길이 `K` 이상인 증가 경로가 존재하는가?"**

이렇게 바꾸면 `L`이 작을수록 조건이 빡빡하고, `L`이 커질수록 조건이 완화되므로 **이분 탐색**이 가능해집니다.

* `L`이 가능하면 더 큰 값도 항상 가능
* `L`이 불가능하면 더 작은 값도 항상 불가능

즉, `False False False True True ...` 형태의 **단조성**이 존재합니다.

### 2) Step 1: `can(limit)` 판별 함수 만들기
이분 탐색의 핵심은 다음 판별 함수를 정확히 만드는 것입니다.

* **의미:**  
  `can(limit)` =  
  **"높이 차가 `limit` 이하인 이동만 허용했을 때, 길이 `K` 이상인 증가 경로가 하나라도 존재하는가?"**

이때 이동 조건은 다음 두 가지를 모두 만족해야 합니다.

1. 다음 칸의 높이가 더 커야 함
2. 높이 차가 `limit` 이하여야 함

if grid[nr][nc] > grid[r][c] and grid[nr][nc] - grid[r][c] <= limit:
