# [HSAT] 슈퍼컴퓨터 클러스터

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 가장 작은 컴퓨터 하나만 올리는 문제가 아님
이 문제의 목표는 어떤 한 컴퓨터의 성능을 크게 만드는 것이 아니라,  
**업그레이드 이후 전체 컴퓨터들의 최저 성능을 최대화하는 것**이다.

즉, 후보 최저 성능을 `x`라고 정했을 때

- 이미 `x` 이상인 컴퓨터는 그대로 두고
- `x`보다 작은 컴퓨터는 **모두 최소한 `x`까지** 올려야 한다.

따라서 판별 함수에서는 `a[i] < x`인 모든 컴퓨터를 대상으로 비용을 계산해야 한다.

---

### 2) `x 이상`이 아니라 `딱 x`까지 올리는 것이 최소 비용
후보 최저 성능이 `x`일 때,  
어떤 컴퓨터를 `x`보다 더 높게 올려도 **최솟값 판정에는 도움이 없고 비용만 더 든다.**

따라서 `a[i] < x`인 컴퓨터는 항상

- 증가량: `x - a[i]`
- 비용: `(x - a[i])^2`

로 계산하면 된다.

즉, `x`를 만들기 위한 총 최소 비용은 다음과 같다.

\[
\sum_{a_i < x} (x-a_i)^2
\]

---

### 3) "정답 구하기"가 아니라 "이 값이 가능한가?"를 판별하는 문제
이 문제는 최적값을 직접 찾기보다,  
다음과 같은 **판별 문제**로 바꿔서 생각하는 것이 핵심이다.

- "최저 성능을 `x`로 만들 수 있는가?"

이 질문에 답할 수 있으면,  
가능한 `x` 중 최댓값을 찾는 문제로 바뀐다.

즉 `can(x)`를 다음처럼 정의하면 된다.

- `True`: 예산 `B` 이하로 모든 컴퓨터를 `x` 이상으로 만들 수 있음
- `False`: 예산을 초과하므로 불가능

---

### 4) 이분탐색은 배열이 아니라 "정답 후보값 x"에 대해 수행
