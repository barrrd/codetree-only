## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 문자열 자체를 만드는 문제가 아니라, **좋은 염기서열들을 몇 개의 그룹으로 묶을지** 결정하는 문제

처음 보면 `.`(와일드카드)가 있으니 직접 초염기서열 문자열을 만들어야 할 것 같지만,  
실제로 중요한 것은 **문자열을 만드는 것**이 아니라

**어떤 좋은 염기서열들끼리 하나의 초염기서열로 동시에 커버 가능한가**를 판단하는 것이다.

즉 이 문제의 핵심은:

- 좋은 염기서열 여러 개를 **한 그룹**으로 묶을 수 있는지 판별
- 전체 `N`개의 좋은 염기서열을 **최소 몇 개의 그룹**으로 나눌 수 있는지 계산

이다.

따라서 문제를 다음 두 단계로 나누어야 한다.

1. 어떤 부분집합이 **초염기서열 1개로 커버 가능한지** 판별
2. 전체를 그런 부분집합들로 **최소 분할**하는 DP 수행

---

### 2) 각 문자를 비트마스크로 바꾸면 “공통 가능 문자” 판별이 쉬워진다

각 위치에는 `a`, `c`, `g`, `t` 중 하나가 올 수 있다.  
이를 비트로 표현하면 다음과 같다.

- `a = 0001`
- `c = 0010`
- `g = 0100`
- `t = 1000`
- `. = 1111`

여기서 `.`은 어떤 문자도 가능하므로 `1111`로 두면 된다.

이제 여러 좋은 염기서열을 동시에 만족시킬 수 있는지 보려면,  
각 위치별로 가능한 문자들의 **교집합**이 비어 있지 않은지 확인하면 된다.

즉 어떤 열에서:

- `a` 와 `.` → `0001 & 1111 = 0001` → 가능
- `a` 와 `c` → `0001 & 0010 = 0000` → 불가능

처럼 비트 AND 연산으로 공통 가능한 문자를 바로 구할 수 있다.

---

### 3) 어떤 부분집합 `mask`가 한 번에 가능한지(`valid[mask]`) 먼저 전처리해야 한다

`mask`는 좋은 염기서열들의 부분집합을 뜻한다.

예를 들어 `N = 4`일 때:

- `0001` → 0번만 포함
- `0101` → 0번, 2번 포함
- `1111` → 전부 포함

이때 `valid[mask]`를 다음 의미로 정의한다.

- `valid[mask] = True`
  - `mask`에 속한 좋은 염기서열들을 **초염기서열 1개로 동시에 커버 가능**
- `valid[mask] = False`
  - 불가능

판정 방법은 단순하다.

각 열 `col`에 대해:

1. 처음엔 `curr = 1111`
2. `mask`에 포함된 sequence들의 `col`번째 문자 비트와 계속 AND
3. 중간에 `curr == 0`이 되면 그 열에서 공통 가능한 문자가 없으므로 실패

즉 모든 열에서 공통 가능한 문자가 하나 이상 존재해야만 `valid[mask] = True`가 된다.

### 4) 이 문제의 DP 상태는 “몇 번째 문자열까지”가 아니라 **어떤 문자열들을 이미 커버했는가**이다

평소 DP처럼 `dp[i]`, `dp[i][j]` 형태로 잡으려고 하면 잘 안 보인다.  
이 문제는 문자열의 순서나 위치가 핵심이 아니라,  
**좋은 염기서열들을 어떤 묶음으로 나눌 것인가**가 핵심이기 때문이다.

따라서 DP를 다음처럼 정의해야 한다.

- `dp[mask] = mask에 해당하는 좋은 염기서열들을 모두 커버하는 데 필요한 최소 초염기서열 개수`

예를 들어 `N = 4`라면:

- `dp[0000] = 0` → 아무 것도 안 덮음
- `dp[0101] = 1` → 0번, 2번을 초염기서열 1개로 커버 가능
- `dp[1111]` → 전체를 커버하는 최소 개수

즉 이 문제는 **위치 DP가 아니라 부분집합 DP**로 풀어야 한다.

---

### 5) `valid[mask] = True`이면 그 집합은 답이 바로 1이다

어떤 `mask`가 이미 초염기서열 1개로 가능하다면,  
그 집합을 덮는 최소 개수는 당연히 `1`이다.

즉:

- `valid[mask] = True` 이면
- `dp[mask] = 1`

이다.

이 경우 더 잘게 쪼개 봐도 최소값이 1보다 작아질 수 없으므로  
추가 분할 탐색은 할 필요가 없다.

---

### 6) 한 번에 덮을 수 없다면, 부분집합으로 쪼개서 최소 분할을 구해야 한다

`valid[mask] = False`이면 `mask` 전체를 초염기서열 1개로는 못 덮는다는 뜻이다.  
그럼 `mask`를 두 부분으로 쪼개야 한다.

즉 어떤 부분집합 `sub ⊆ mask`를 골라서

- `sub`
- `mask ^ sub`

로 분할하고,  
두 집합을 각각 덮는 최소값을 더한 것 중 최소를 취한다.

점화식은 다음과 같다.

\[
dp[mask] = \min(dp[sub] + dp[mask \setminus sub])
\]

비트마스크 구현에서는 `mask ^ sub`가 `mask \setminus sub` 역할을 한다.

즉:

- `sub`는 `mask`의 부분집합
- `mask ^ sub`는 나머지 부분

이다.
