# [HSAT 기출] 도로 보수 로봇

> **문제 링크**: [코드트리 - 도로 보수 로봇](https://www.codetree.ai/ko/frequent-problems/hsat/problems/road-repair-robot/description)

---

## ⚠️ 해결 시 주의해야 할 핵심 포인트

### 1) 핵심 아이디어: 파라메트릭 서치 (Parametric Search)
이 문제는 "패치의 최소 길이는 얼마인가?"라는 질문을 **"길이 $L$인 패치 $K$개로 모든 구멍을 막을 수 있는가?"**라는 결정 문제로 바꾸어 해결해야 하는 전형적인 이분 탐색 유형입니다.

* **핵심:** 구멍의 위치($X$)와 패치의 개수($K$)가 최대 $10^9$로 매우 크기 때문에, 단순 브루트 포스가 아닌 로그 단위($\log 10^9$)의 효율적인 탐색이 필수적입니다.
* **➡️ 해결 방법:** 이분 탐색을 통해 패치 길이를 조절하며 최적의 해(최소 길이)를 찾아나갑니다.

### 2) Step 1: 그리디(Greedy)하게 패치 배치하기
특정 길이 `mid_l`이 주어졌을 때, 패치를 최소한으로 사용하여 모든 구멍을 막는 로직이 가장 중요합니다.

* **오류 예시:** 구멍의 중간 지점부터 패치를 붙이거나, 패치 사이의 간격을 복잡하게 계산하여 예외 상황을 만드는 경우입니다.
* **핵심:** **가장 왼쪽에 있는 막히지 않은 구멍**을 기준으로 패치의 **시작점**을 잡는 것이 언제나 최적의 선택입니다.
* **➡️ 해결 방법:**
    ```python
    fin_point = -1 # 마지막으로 커버된 위치 (초기값)
    cnt = 0
    for p in positions:
        if p > fin_point: # 현재 구멍이 기존 패치 범위를 벗어났다면
            cnt += 1 # 새 패치 사용
            fin_point = p + mid_l - 1 # 현재 구멍부터 mid_l만큼 커버
    ```

### 3) Step 2: 이분 탐색의 방향 설정 (최솟값 찾기)
조건을 만족하는 값 중 **가장 작은 값**을 찾아야 하므로, 성공(True)과 실패(False)에 따른 범위 업데이트 방향을 정확히 설정해야 합니다.

* **오류 예시:** `cnt <= k`일 때(성공했을 때) `min_l`을 키우는 경우입니다. 이 경우 최솟값이 아닌 '성공 가능한 최대값' 쪽으로 수렴하게 됩니다.
* **핵심:** 성공했다면 **"더 짧은 길이로도 가능한가?"**를 확인하기 위해 상한선(`max_l`)을 줄여야 합니다.
* **➡️ 해결 방법:**
    * `if cnt <= k:` (성공): `answer = mid_l`, `max_l = mid_l - 1` (왼쪽 탐색)
    * `else:` (실패): `min_l = mid_l + 1` (오른쪽 탐색)

### 4) Step 3: 데이터 정렬 및 입력 조건 확인
문제 조건에서 구멍의 위치가 **오름차순**으로 주어진다고 명시되어 있으므로 별도의 정렬은 필요 없으나, 만약 정렬되지 않은 입력이 들어온다면 반드시 `sort()`를 수행해야 그리디 로직이 작동합니다.
