# [HSAT] 동전 프로모션

> **문제 링크**: [코드트리 - 동전 프로모션](https://www.codetree.ai/ko/frequent-problems/hsat/problems/coin-promotion/description)

---

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 그리디(Greedy)의 함정: "가장 큰 동전부터"는 정답이 아님
동전 거스름돈 문제에서 동전들이 서로 배수 관계(예: 10원, 50원, 100원)가 아닐 경우, 무조건 큰 금액부터 채우는 방식은 최적해를 보장하지 않습니다.

* **오류 예시:** 8원을 만들 때 5원(A), 4원(A), 1원(A)이 있다면, 그리디는 `5+1+1+1` (4개)를 선택하지만 최적해는 `4+4` (2개)입니다.
* **➡️ 해결 방법:** 모든 가능한 조합을 효율적으로 탐색하여 최소값을 저장하는 **DP(Dynamic Programming)**를 사용해야 합니다.

---

### 2) A 타입(무제한): 순방향(Forward) 업데이트
같은 종류의 동전을 여러 번 사용할 수 있는 A 타입은 **앞에서부터** 채워나가야 합니다.

* **핵심:** `dp[i]`를 구할 때, **방금 이번 루프에서 업데이트된** `dp[i - value]`를 참조할 수 있어야 합니다.
* **논리:** "이미 이 동전을 하나 쓴 상태"에서 똑같은 동전을 한 번 더 얹는 계산이 가능해지므로 자연스럽게 무제한 복제가 허용됩니다.
* **➡️ 구현 방법:** `for i in range(value, m + 1):`

---

### 3) B 타입(한정판): 역방향(Backward) 업데이트
각 종류별로 한 번만 사용할 수 있는 B 타입은 반드시 **뒤에서부터** 거꾸로 계산해야 합니다.

* **핵심:** `dp[i]`를 구할 때 참조하는 `dp[i - value]`가 **아직 이번 동전의 영향을 받지 않은 "이전 단계의 값"**이어야 합니다.
* **논리:** 뒤에서부터 계산하면 내가 참조할 앞쪽 칸들이 아직 갱신되기 전이므로, 동전이 중복해서 사용되는 것을 완벽하게 차단합니다.
* **➡️ 구현 방법:** `for i in range(m, value - 1, -1):`

---

### 4) 인덱스 범위와 초기값 설정 (Runtime Error 방지)
DP 배열을 다룰 때 가장 흔하게 발생하는 `Index Error`를 방지해야 합니다.

* **범위 설정:** 역방향 루프 시 `i - value`가 `0`이 되는 시점까지 포함해야 하므로, range의 끝값은 `value - 1`로 설정해야 합니다.
* **초기화:** `dp[0] = 0` (0원을 만드는 데 필요한 동전은 0개)으로 시작하고, 나머지 칸은 아주 큰 값(`float('inf')`)으로 채워 "아직 만들 수 없음"을 표시합니다.

---

### 5) 1차원 배열로 메모리 최적화
원래는 2차원 배열(`dp[동전종류][금액]`)이 정석이지만, 탐색 방향(순방향/역방향)만 조절하면 **단 한 줄의 리스트**만으로도 무제한/한정판 동전의 로직을 모두 구현할 수 있습니다.

---

## 풀이 핵심 요약

이 문제는 **1차원 냅색(Knapsack) DP**를 활용하여 동전의 타입에 따라 업데이트 방향을 결정하는 문제입니다.

1. `dp` 배열을 `inf`로 초기화하고 `dp[0] = 0`으로 설정한다.
2. 동전 목록을 순회하며 타입(`A` 또는 `B`)을 확인한다.
3. **A 타입**이면 앞에서부터(`value` → `m`) 훑으며 `dp[i] = min(dp[i], dp[i-value] + 1)`을 수행한다.
4. **B 타입**이면 뒤에서부터(`m` → `value`) 훑으며 동일한 점화식을 적용한다.
5. 최종적으로 `dp[m]`이 여전히 `inf`라면 거슬러 줄 수 없는 경우이므로 `-1`을 출력한다.

---

## 사용한 알고리즘 및 문법

- **DP (동적 계획법):** 작은 문제의 해를 저장하여 큰 문제 해결 (1차원 냅색)
- **순방향/역방향 탐색:** 중복 허용 여부에 따른 포인터 이동 전략
- **zip():** 동전의 금액 리스트와 타입 리스트를 쌍으로 묶어 처리
- **인덱스 제어:** `range(start, stop, step)`을 활용한 정밀한 배열 접근
