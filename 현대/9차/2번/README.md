# [HSAT] 채광 로봇 시뮬레이션

> **문제 링크**: [코드트리 - 채광 로봇 시뮬레이션](https://www.codetree.ai/ko/frequent-problems/hsat/problems/mining-robot-simulation/description)

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 시간 역행을 “현재 위치”가 아니라 “되돌아갈 기준 위치”로 바라봐야 함
이 문제에서 가장 헷갈리는 부분은 시간 역행 장치를 사용하는 시점을 어떻게 해석하느냐입니다.

* **오류 예시:** `(r, c)`에서 장치를 사용한다고 생각하고, 단순히 그 위치까지의 최대 이득 + 이후 경로만 붙이려고 하면, “정확히 T초 전으로 이동”하는 구조를 반영하지 못합니다.
* **핵심:** 실제로는 어떤 칸 `(r, c)`를 **되돌아갈 기준 위치**로 잡고,  
  그 위치에서 **T초 더 이동한 뒤** 장치를 사용하여 다시 `(r, c)`로 돌아온다고 해석해야 합니다.
* **➡️ 해결 방법:**
    * 전체 경로를 다음 3구간으로 분리합니다.
        1. 시작점 `(0,0)` → `(r,c)`
        2. `(r,c)`에서 정확히 `T`초 동안 더 이동
        3. 시간 역행 후 다시 `(r,c)` → 도착점 `(n-1,n-1)`
    * 따라서 최종 이득은 단순 경로 복원이 아니라, **세 구간의 최대값을 DP로 결합**하는 방식으로 접근해야 합니다.

---

### 2) 경로 자체를 복원하려고 하면 안 되고, 값만 관리하는 DP로 풀어야 함
오른쪽 / 아래로만 이동하므로 얼핏 보면 최적 경로를 하나 복원해서 시간 역행 위치를 찾을 수 있을 것처럼 보이지만, 실제로는 최적 경로가 여러 개 존재할 수 있습니다.

* **오류 예시:** `dp[r][c]`를 만든 뒤, 도착점에서 시작해서 위 / 왼쪽 중 값이 더 큰 방향으로 역추적하며 유일한 최적 경로를 찾으려는 방식.
* **핵심:** 같은 최대 이득을 만드는 경로가 여러 개 존재할 수 있으므로, 특정 경로 하나를 복원해서 처리하면 오답이 날 수 있습니다.
* **➡️ 해결 방법:**
    * 경로를 직접 구하지 말고, 각 구간의 **최대 이득만 저장하는 DP**를 사용합니다.
    * 즉,
        * `forward[r][c]` : 시작점 → `(r,c)` 최대 이득
        * `backward[r][c]` : `(r,c)` → 도착점 최대 이득
        * `extra[r][c]` : `(r,c)`에서 정확히 `T`초 더 이동하며 얻는 최대 추가 이득
      으로 분리해서 계산합니다.

---

### 3) `forward DP`와 `backward DP`를 분리해서 생각해야 함
시간 역행을 기준 위치 `(r,c)`에서 수행한다고 보면, 앞쪽 구간과 뒤쪽 구간을 따로 최적화해야 합니다.

* **오류 예시:** 시작점에서 각 칸까지의 DP 하나만 만든 뒤, 그 값을 이용해서 뒤쪽 경로까지 한 번에 처리하려는 방식.
* **핵심:**  
  시작점 → `(r,c)` 와 `(r,c)` → 도착점은 서로 다른 방향의 DP입니다.  
  앞은 **위 / 왼쪽에서 오는 최대값**, 뒤는 **아래 / 오른쪽으로 가는 최대값**을 각각 따로 계산해야 합니다.
* **➡️ 해결 방법:**
    * `forward[r][c]`
      ```python
      forward[r][c] = max(forward[r-1][c], forward[r][c-1]) + grid[r][c]
      ```
    * `backward[r][c]`
      ```python
      backward[r][c] = max(backward[r+1][c], backward[r][c+1]) + grid[r][c]
      ```
    * `forward`는 `(0,0)`부터 정방향 순회,
      `backward`는 `(n-1,n-1)`부터 역방향 순회로 계산합니다.

---

### 4) 시간 역행 전 “정확히 T초 동안 더 움직이는 구간”은 `prev / curr` 형태의 추가 DP로 계산해야 함
이 문제의 핵심은 시간 역행을 쓰기 전에, 어떤 기준 위치 `(r, c)`에서 **정확히 `T`번 더 이동하며 얻을 수 있는 최대 추가 이득**을 구하는 것입니다.

* **오류 예시:** `forward[r][c] + backward[r][c]`만 계산하고, 시간 역행 직전 `T`초 동안 더 진행하며 얻는 이득을 누락하는 경우.
* **핵심:**  
  `(r, c)`를 **시간 역행 후 되돌아올 기준 위치**로 두고,  
  거기서 시작해서 **정확히 `T`번 이동했을 때 얻을 수 있는 최대 추가 이득**을 별도로 계산해야 합니다.

* **➡️ 해결 방법:**
    * `prev[r][c]`를  
      **“현재 위치 `(r, c)`에서 시작해서, 정확히 `k`번 이동하며 얻을 수 있는 최대 추가 이득”**  
      이라고 두고, 이동 횟수를 1씩 늘려가며 갱신합니다.
    * 초기 상태는 `k = 0` 이므로
      ```python
      prev = [[0] * n for _ in range(n)]
      ```
      로 둡니다.  
      즉, **0번 이동하면 추가 이득은 0**입니다.
    * 이후 매 단계마다 `curr`를 새로 만들어,
      * 아래로 1칸 이동하는 경우
      * 오른쪽으로 1칸 이동하는 경우
      를 비교하여 최댓값을 채웁니다.
    * 점화식은 코드상 다음과 같은 의미입니다.
      ```python
      curr[r][c] = max(
          prev[r + 1][c] + grid[r + 1][c],   # 아래로 이동
          prev[r][c + 1] + grid[r][c + 1]    # 오른쪽으로 이동
      )
      ```
    * 즉, `(r, c)`에서 한 칸 이동해 새로 밟는 칸의 값을 얻고,  
      남은 이동 횟수는 그 다음 칸에서의 최적값(`prev`)을 이어붙이는 구조입니다.

* **구현 형태:**
    ```python
    def step3():
        INF = -float("inf")

        # 0번 이동했을 때의 추가 이득
        prev = [[0] * n for _ in range(n)]

        for _ in range(t):
            curr = [[INF] * n for _ in range(n)]

            for r in range(n - 1, -1, -1):
                for c in range(n - 1, -1, -1):

                    if r < n - 1 and prev[r + 1][c] != INF:  # go down
                        curr[r][c] = max(curr[r][c], prev[r + 1][c] + grid[r + 1][c])

                    if c < n - 1 and prev[r][c + 1] != INF:  # go right
                        curr[r][c] = max(curr[r][c], prev[r][c + 1] + grid[r][c + 1])

            prev = curr

        return prev
    ```

* **최종 의미:**  
  `step3()`의 반환값 `extra[r][c]`는  
  **“현재 위치 `(r,c)`에서 시작해서, 정확히 `T`번 이동하며 새로 얻을 수 있는 최대 추가 이득”**  
  을 뜻합니다.  
  단, **출발 칸 `(r,c)` 자체의 값은 포함하지 않고**, 이동해서 새로 밟는 칸들의 값만 더합니다.

### 5) `extra`에서 도착점 `(n-1, n-1)`은 `T > 0`일 때 0이 아니라 불가능 상태(`-INF`)가 되어야 함
`extra DP`를 구현할 때 자주 헷갈리는 포인트입니다.

* **오류 예시:** 도착점에서는 더 이상 이동할 수 없으므로 `extra[n-1][n-1] = 0`으로 계속 유지하는 경우.
* **핵심:** `extra[r][c]`는 **정확히 `T`번 이동한 결과**를 의미합니다.  
  따라서 `T > 0`인데 더 갈 수 있는 칸이 없다면, 그 상태는 **0점이 아니라 아예 불가능한 상태**여야 합니다.
* **➡️ 해결 방법:**
    * `k = 0`일 때만 모든 칸에서 `prev[r][c] = 0`
    * `k >= 1`부터는 이동 가능한 칸만 `curr[r][c]`가 갱신되고,
      이동할 수 없는 칸은 계속 `-INF`로 남도록 둡니다.
    * 따라서 `(n-1, n-1)`은
        * `0번 이동`이면 `0`
        * `1번 이상 이동`이면 `-INF`
      가 맞습니다.

---

### 6) 시간 역행을 “안 쓰는 경우”는 도착점 값 하나만 비교해야 함
시간 역행을 쓰지 않는 경우도 정답 후보에 포함해야 하지만, 중간 칸의 DP 값 전체를 후보로 넣으면 안 됩니다.

* **오류 예시:** `forward[r][c]` 전체를 순회하며 시간 역행을 안 쓴 경우의 후보로 함께 비교하는 방식.
* **핵심:** 문제는 최종적으로 반드시 **도착점 `(n-1,n-1)`에 도달**해야 하므로, 시간 역행을 사용하지 않는 경우의 값은 오직 `forward[n-1][n-1]` 하나뿐입니다.
* **➡️ 해결 방법:**
    * 정답 초기값을
      ```python
      answer = forward[n-1][n-1]
      ```
      로 둡니다.
    * 그 후 시간 역행을 사용하는 모든 기준 위치 `(r,c)`에 대해
      ```python
      answer = max(answer, forward[r][c] + extra[r][c] + backward[r][c])
      ```
      를 비교합니다.
    * 단, `extra[r][c] == -INF` 인 경우는 정확히 `T`초 더 이동할 수 없는 위치이므로 제외합니다.

---

## 풀이 핵심 요약

이 문제는 단순 경로 복원 문제가 아니라,  
**시간 역행 전후의 경로를 구간별 DP로 분리해서 결합하는 문제**입니다.

핵심 아이디어는 다음과 같습니다.

1. `forward DP`로 시작점 → 각 칸까지의 최대 이득 계산
2. `backward DP`로 각 칸 → 도착점까지의 최대 이득 계산
3. `extra DP`로 각 칸에서 정확히 `T`초 더 이동할 때의 최대 추가 이득 계산
4. 모든 칸 `(r,c)`를 시간 역행 기준 위치로 보며  
   `forward[r][c] + extra[r][c] + backward[r][c]` 의 최댓값을 구함
5. 시간 역행을 전혀 사용하지 않는 경우 `forward[n-1][n-1]` 와도 비교

---

## 사용한 알고리즘

- DP (동적 계획법)
- 정방향 DP
- 역방향 DP
- 시간 길이(`T`)를 축으로 한 추가 DP
