# [HSAT] 사물인식 최소 면적 산출 프로그램

> **문제 링크**: [코드트리 - 사물인식 최소 면적 산출 프로그램](https://www.codetree.ai/ko/frequent-problems/hsat/problems/minimum-area-for-object-recognition/description)

---

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 전체 점들 중 K개를 아무렇게나 고르는 조합 문제가 아님

이 문제는 전체 점들 중에서 K개를 고르는 일반적인 조합 문제가 아니다.
핵심은 **각 색깔 그룹에서 정확히 1개씩 대표 점을 선택하는 것**이다.

즉 다음과 같이 생각해야 한다.

* 1번 색에서 점 1개 선택
* 2번 색에서 점 1개 선택
* ...
* K번 색에서 점 1개 선택

그리고 이렇게 고른 점들을 모두 포함하는 최소 직사각형의 넓이를 계산해야 한다.

따라서 이 문제는
**전체 점 중 K개를 고르는 combinations** 보다는,
**색깔별 리스트에서 하나씩 꺼내는 DFS / 백트래킹 탐색**으로 보는 것이 맞다.

---

### 2) 모든 경우를 `cases` 리스트에 저장하면 메모리 초과가 발생하기 쉽다

처음에는 색별로 하나씩 뽑은 모든 경우를 다음처럼 만들고 싶어진다.

```python
cases = [[]]

for color in colors:
    new_cases = []

    for case in cases:
        for point in points[color]:
            new_cases.append(case + [point])

    cases = new_cases
```

이 방식은 논리적으로는 맞지만,
경우의 수가

```python
len(points[c1]) * len(points[c2]) * ... * len(points[cK])
```

형태로 커지기 때문에 `cases` 자체가 매우 커질 수 있다.

즉, 완성된 조합 전체를 저장하는 방식은 메모리 낭비가 심하다.
그래서 이 문제는 **현재 상태만 들고 내려가는 DFS / 백트래킹**으로 풀어야 한다.

---

### 3) DFS에서 `path` 전체를 들고 다니지 말고, 직사각형 범위만 관리해야 한다

처음 떠올리기 쉬운 DFS는 이런 형태다.

```python
def dfs(idx, path):
    ...
```

여기서 `path`는 지금까지 선택한 점들의 리스트이다.

하지만 이 방식은 최종 단계마다 `path` 전체를 다시 순회해서

* `min_x`
* `max_x`
* `min_y`
* `max_y`

를 계산해야 하므로 비효율적이다.

이 문제에서 실제로 필요한 것은 선택한 점들 전체가 아니라,
그 점들로 만들어지는 **현재 직사각형의 범위**뿐이다.

따라서 DFS 상태는 다음 5개만 넘기면 충분하다.

* `idx`: 현재 몇 번째 색을 처리 중인지
* `min_x`
* `max_x`
* `min_y`
* `max_y`

즉, `path`를 직접 넘기는 대신
**직사각형 상태를 압축해서 넘기는 DFS**가 핵심이다.

---

### 4) `idx`의 의미: 현재 몇 번째 색을 처리 중인가

DFS에서 가장 먼저 이해해야 할 인자는 `idx`이다.

```python
def dfs(idx, min_x, max_x, min_y, max_y):
```

여기서 `idx`는
**현재 `colors[idx]`에 해당하는 색을 처리할 차례**라는 뜻이다.

예를 들어 `colors = [3, 1, 2]` 라면,

* `idx = 0` → 3번 색 처리
* `idx = 1` → 1번 색 처리
* `idx = 2` → 2번 색 처리

가 된다.

즉 `idx`는 단순한 인덱스이지만,
실제로는 **색 선택 진행 단계**를 의미한다.

---

### 5) 첫 점도 일반 갱신식으로 처리할 수 있도록 초기값을 잘 잡아야 한다

DFS를 시작할 때는 아직 어떤 점도 고르지 않은 상태이므로
직사각형 범위는 다음처럼 초기화하면 된다.

```python
dfs(0, float("inf"), -float("inf"), float("inf"), -float("inf"))
```

그러면 첫 점 `(x, y)`를 넣었을 때도 일반 갱신식만으로 처리할 수 있다.

```python
nmin_x = min(min_x, x)
nmax_x = max(max_x, x)
nmin_y = min(min_y, y)
nmax_y = max(max_y, y)
```

초기값이 `inf`, `-inf` 이므로 첫 점이 들어가면 자동으로

* `min_x = x`
* `max_x = x`
* `min_y = y`
* `max_y = y`

가 된다.

즉, 따로 `if idx == 0` 같은 분기를 두지 않아도
모든 단계에서 같은 갱신 로직을 사용할 수 있다.

---

### 6) DFS 인자 4개는 지금까지 만든 직사각형의 요약 정보다

처음 DFS를 보면

```python
dfs(idx, min_x, max_x, min_y, max_y)
```

처럼 인자가 많아서 헷갈리기 쉽다.

하지만 의미를 풀어보면 단순하다.

* `idx`: 몇 번째 색인지
* `min_x, max_x, min_y, max_y`: 지금까지 선택한 점들로 만든 직사각형 상태

즉, 이 네 값은 사실
아까 리스트로 관리하던 `path`의 **압축본**이다.

원래 `path = [(x1, y1), (x2, y2), ...]` 전체를 들고 다녔지만,
직사각형 넓이를 계산할 때 실제로 필요한 것은
그 점들의 최소/최대 좌표뿐이다.

따라서 DFS 인자에 무엇을 넘길지 고민될 때는
항상 **다음 단계로 가는 데 꼭 필요한 정보만 넘긴다**고 생각하면 된다.

---

### 7) 현재 점을 추가할 때는 직사각형 범위를 O(1)로 갱신한다

새 점 `(x, y)`를 선택했을 때 직사각형 범위는 다음처럼 갱신한다.

```python
nmin_x = min(min_x, x)
nmax_x = max(max_x, x)
nmin_y = min(min_y, y)
nmax_y = max(max_y, y)
```

이 갱신은 **현재 직사각형에 점 하나를 추가하는 것**이다.

이렇게 하면 매 단계마다 O(1)로 상태를 갱신할 수 있고,
마지막에 전체 `path`를 다시 순회할 필요가 없다.

즉, 이 문제가 DFS로 풀릴 때 중요한 이유는
단순히 재귀라서가 아니라,
**직사각형 상태를 점진적으로 누적하면서 탐색할 수 있기 때문**이다.

---

### 8) 가지치기(Pruning)가 사실상 시간 단축의 핵심이다

현재까지 만든 직사각형 넓이가 이미 `answer` 이상이라면,
앞으로 점을 더 추가해도 직사각형 넓이는 절대 줄어들지 않는다.

따라서 다음과 같은 가지치기가 가능하다.

```python
area = (nmax_x - nmin_x) * (nmax_y - nmin_y)
if area >= answer:
    continue
```

이 조건이 중요한 이유는,
현재 상태만 봐도 이미 최적해보다 나쁜 경우라면
그 아래 경우 전체를 볼 필요가 없기 때문이다.

즉, DFS 자체보다도
**DFS + 가지치기**가 이 문제를 통과시키는 핵심이다.

---

### 9) 색 순서를 점 개수가 적은 색부터 정렬하면 더 유리하다

색을 무조건 `1, 2, 3, ..., K` 순서로 탐색할 필요는 없다.

오히려 다음처럼 각 색의 점 개수가 적은 순서대로 정렬하면 좋다.

```python
colors = list(range(1, K + 1))
colors.sort(key=lambda c: len(points[c]))
```

이유는 DFS 초반 분기 수를 줄일 수 있기 때문이다.

예를 들어

* 1번 색 점 20개
* 2번 색 점 2개
* 3번 색 점 3개

라면, 점이 적은 색부터 먼저 고르는 것이 훨씬 유리하다.
초반에 분기 수가 작아야 가지치기도 더 빠르게 걸릴 가능성이 커진다.

즉, 이 정렬은 정답을 바꾸는 것은 아니지만
탐색 효율을 크게 높여준다.

---

## DFS가 이 문제에서 맞는 이유

이 문제는 그래프를 방문하는 전형적인 DFS와는 다르다.
여기서의 DFS는 **색 그룹별로 하나씩 선택하는 모든 경우를 깊이 우선으로 내려가는 백트래킹 탐색**이다.

흐름은 다음과 같다.

1. 현재 처리할 색 `colors[idx]`를 정한다.
2. 그 색에 있는 모든 점을 하나씩 선택해본다.
3. 점 하나를 선택할 때마다 직사각형 범위를 갱신한다.
4. 현재 넓이가 이미 `answer` 이상이면 중단한다.
5. 모든 색을 다 선택하면 넓이를 계산해 최소값을 갱신한다.

즉, 아까 `cases = [[]]` 방식으로 전체 경우를 계속 만들던 것을
DFS에서는 **저장하지 않고 하나의 경우를 따라 내려가며 처리**하는 것이다.

그래서 메모리도 줄고, 가지치기도 가능해진다.

---

## 풀이 핵심 요약

이 문제는 **색별 좌표를 그룹화한 뒤, 각 색에서 점 하나씩 선택하는 DFS / 백트래킹 + 가지치기** 문제이다.

1. 입력을 받아 `points[color]` 형태로 색별 좌표를 저장한다.
2. `colors`를 점 개수 기준으로 정렬한다.
3. DFS 상태를 `idx, min_x, max_x, min_y, max_y` 로 둔다.
4. 현재 색의 모든 점을 하나씩 시도하며 직사각형 범위를 갱신한다.
5. 현재 넓이가 이미 `answer` 이상이면 가지치기한다.
6. 모든 색을 다 선택하면 넓이를 계산해 최소값을 갱신한다.

---

## 사용한 알고리즘 및 문법

* **DFS / 백트래킹:** 각 색에서 점 하나씩 고르는 모든 경우 탐색
* **가지치기(Pruning):** 현재 넓이가 이미 최적해 이상이면 중단
* **리스트 그룹화:** `points[color].append((x, y))`
* **정렬:** `colors.sort(key=lambda c: len(points[c]))`
* **최소/최대 갱신:** `min()`, `max()`를 이용한 직사각형 범위 관리
* **재귀 상태 압축:** `path` 대신 `min_x, max_x, min_y, max_y` 사용
