# [삼성 기출] 미생물 연구

> **문제 링크**: [코드트리 - 미생물 연구](https://www.codetree.ai/ko/frequent-problems/samsung-sw/problems/microbial-research/description)

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 파먹힌 미생물의 타이트한 경계(Bounding Box) 재계산
새로운 배양 용기로 미생물을 옮길 때, 다른 미생물에게 먹혀 모양이 줄어든 미생물의 **실제 크기(여백을 제외한 타이트한 경계)**를 다시 구해서 모양(`shape`)을 떠야 합니다.

* **오류 예시:** 처음 투입됐을 때의 커다란 가로세로 길이(`rlen`, `clen`)와 배열 크기를 그대로 유지하여 `shape`를 만들면, 미생물 주변에 '투명한 여백(False)'이 껍데기처럼 남게 됩니다. 이로 인해 새 용기의 벽면(`[0][0]`)에 바짝 붙어서 배치되지 못하고 허공에 붕 뜨는 논리적 오류가 발생합니다.
* **핵심:** 현재 격자 내에 살아남은 미생물의 `True` 칸들을 기준으로 가장 끝에 있는 최소/최대 좌표를 새롭게 갱신해야 합니다.
* **➡️ 해결 방법:**
    * 미생물을 탐색하며 살아남은 칸(`arr[r][c] == k`)의 `min_r, max_r, min_c, max_c`를 구합니다.
    * 갱신된 좌표를 바탕으로 `t_rlen = max_r - min_r + 1`, `t_clen = max_c - min_c + 1` 크기의 타이트한 2차원 배열 `shape`를 새롭게 선언합니다.
    * `shape[r - min_r][c - min_c] = True` 방식으로 여백 없이 꽉 찬 미생물 도장을 완성합니다.

### 2) '유령 미생물' (완전 소멸 및 배치 실패) 완벽 삭제
문제의 조건에 따라 '사라져야 하는' 미생물들이 격자(`arr`)에서는 지워지지만, 관리 딕셔너리(`order_dct`)에 계속 남아 다음 턴에 부활하거나 에러를 유발하는 현상을 차단해야 합니다.

* **오류 예시:** 새로 투입된 미생물에게 100% 잡아먹혀 조각조각 나지도 않은 미생물(`divide_num == 0`)이나, 새 용기에 자리가 없어 쫓겨난 미생물(`possible == False`)을 딕셔너리에서 `del`로 지워주지 않는 실수.
* **핵심:** 삭제 조건을 일일이 분기 처리하는 것보다, **무사히 살아남아 새 집에 안착한 미생물들만 모은 새로운 딕셔너리**를 만들어 통째로 교체하는 것이 훨씬 안전하고 깔끔합니다.
* **➡️ 해결 방법:**
    * Bounding Box를 갱신할 때 `total == 0`인 미생물(100% 잡아먹힌 미생물)은 `continue`로 입구 컷을 시킵니다.
    * 새 용기 배치가 끝난 후, 배치가 완료된(`possible == True`) 미생물의 좌표만 `new_order_dct`에 저장합니다.
    * 턴의 마지막에 `order_dct = new_order_dct`로 덮어씌워 유령 미생물들을 메모리에서 완전히 청소합니다.

### 3) 새 배양 용기 배치 시 탐색 순서 (우선순위)
"최대한 x좌표가 작은 위치, 같다면 y좌표가 작은 위치"라는 문제의 조건을 2차원 배열의 행(`r`)과 열(`c`) 탐색 순서로 정확히 치환해야 합니다.

* **오류 예시:** x좌표를 열(`c`), y좌표를 행(`r`)으로 매핑해 놓고 `for r: for c:` 순서로 탐색할 경우, "y좌표가 작은 위치"를 최우선으로 탐색하게 되어 미생물이 의도치 않게 바닥으로 뚝 떨어지는 현상이 발생합니다.
* **핵심:** 본인이 설계한 좌표계(x가 행인지 열인지)에 따라 이중 `for`문의 바깥쪽 루프를 결정해야 합니다. (작성된 코드는 x축을 행 `r`, y축을 열 `c`로 정의하여 일관성을 맞춘 상태입니다.)
* **➡️ 해결 방법:**
    * x좌표(행, `r`) 최우선 조건을 만족하기 위해 바깥쪽 루프를 `for r in range(n - rlen + 1):`로 설정합니다.
    * 안쪽 루프를 `for c in range(n - clen + 1):`로 두어, 위에서 아래로(`r`), 왼쪽에서 오른쪽으로(`c`) 빈자리를 샅샅이 탐색하도록 보장합니다.

### 4) 인접 미생물 성과 계산 시 BFS 큐(Queue) 삽입 조건 제한
인접한 다른 미생물을 파악하여 성과를 계산할 때, BFS 탐색이 남의 미생물 구역을 타고 넘어가는 전염병 형태가 되지 않도록 제어해야 합니다.

* **오류 예시:** 다음 탐색 칸(`nr, nc`)이 빈칸이 아닐 때 무조건 큐(`q.append`)에 집어넣어, 남의 미생물 영역까지 내 탐색 범위로 간주해버리는 논리적 오류.
* **핵심:** BFS는 오직 **'나와 동일한 미생물 번호'**를 가진 칸으로만 뻗어 나가야 하며, 다른 미생물을 만났을 때는 인접했다는 사실만 기록하고 멈춰야 합니다.
* **➡️ 해결 방법:**
    * 4방향 탐색 중 미생물이 존재하는 칸(`arr[nr][nc] != 0`)을 만났을 때, 번호를 비교합니다.
    * `if arr[nr][nc] == key:` 일 때만 `q.append((nr, nc))`를 수행하여 내 구역을 넓혀갑니다.
    * `else:` (다른 미생물일 경우) 큐에 넣지 않고 `adj[key][arr[nr][nc]] = 1` 로 인접 여부만 체크한 뒤 탐색을 종료(`continue`)합니다.
