# [삼성 기출] 민트 초코 우유

> **문제 링크**: [코드트리 - 민트 초코 우유](https://www.codetree.ai/ko/frequent-problems/samsung-sw/problems/mint-choco-milk/description)

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 파이썬 스코프(Scope) 특성으로 인한 변수명 섀도잉(Shadowing) 에러

2차원 배열을 순회하는 도중, 내부 반복문에서 동일한 변수명을 사용하여 외부 반복문의 인덱스가 오염되는 치명적인 버그가 발생하기 쉽습니다.
- **오류 예시:** 바깥쪽 루프에서 `for r in range(n): for c in range(n):`을 돌고 있는데, 안쪽의 정렬된 리스트를 순회할 때 무의식적으로 `for r, c, _ in sorted_visited[1:]:`를 사용하면 바깥쪽의 `r`, `c` 값이 덮어씌워져 다음 탐색 좌표가 완전히 꼬여버립니다.
- **핵심:** 파이썬은 C/Java와 같은 블록 스코프(`{ }`)가 아니라 **함수 레벨 스코프**를 가지므로, 안쪽 반복문에서 변경된 변수값이 바깥쪽 반복문에도 그대로 영향을 미칩니다.
- **➡️ 해결 방법:**
    - 안쪽 반복문의 튜플 언패킹 변수명을 `for tr, tc, _ in sorted_visited[1:]:` 와 같이 외부 변수와 명확히 분리하여 독립적으로 관리합니다.

### 2) 복잡한 문자열 조합 처리를 위한 비트마스킹(Bitmasking) 도입

'약한 전파' 발생 시 기존 사상과 새로운 사상을 합치고 중복을 제거하는 과정을 문자열로 처리하면 연산이 무겁고 코드가 지저분해집니다.
- **오류 예시:** `set(farr[nr][nc] + ref)` 형태로 문자열을 더하고 중복을 제거한 뒤 `"".join()`으로 다시 합치는 방식은 직관적이지만, 단일/이중/삼중 그룹을 분류할 때 길이나 문자를 일일이 대조해야 하는 번거로움이 있습니다.
- **핵심:** 조합의 경우의 수가 한정적일 때는 각 요소를 **이진수 비트(1, 2, 4)**로 치환하여 관리하는 것이 압도적으로 효율적입니다.
- **➡️ 해결 방법:**
    - 입력 단계부터 민트(1), 초코(2), 우유(4)로 정수 치환합니다.
    - 그룹 분류 시 단일(`1, 2, 4`), 이중(`3, 5, 6`), 삼중(`7`) 조합을 직관적으로 매핑합니다.
    - 약한 전파 시 비트 OR 연산자인 `farr[nr][nc] |= ref` 단 한 줄로 중복 제거와 조합을 동시에 해결합니다.

### 3) BFS 탐색 시 메모리 초과(MLE) 방지를 위한 방문 처리
대표자를 뽑기 위해 인접한 같은 음식의 칸을 탐색할 때, 방문 처리의 위치가 잘못되면 큐(Queue)가 기하급수적으로 팽창합니다.
- **오류 예시:** 큐에서 좌표를 꺼낸 뒤(`popleft`)에 방문 처리를 하거나, 큐에 넣을 때 방문 처리를 누락하면 인접한 두 칸이 서로를 무한히 큐에 집어넣어 메모리가 터집니다.
- **핵심:** BFS에서 중복 탐색을 막기 위해서는 반드시 **"큐에 데이터를 넣는 시점"**에 방문 도장을 찍어야 합니다.
- **➡️ 해결 방법:**
    - `while q:` 내부에서 다음 칸이 조건(`farr[nr][nc] == ref`)을 만족할 때, `q.append()`를 호출하기 직전에 `v[nr][nc] = True`를 선언하여 방문을 확정 짓습니다.

### 4) 턴제 시뮬레이션의 '방어 상태'와 '턴 스킵' 우선순위
전파가 순서대로 진행되며 실시간으로 지도가 업데이트되는 환경에서는, '내가 남을 공격하기 전에 내가 먼저 당했을 경우'를 철저히 통제해야 합니다.
- **오류 예시:** `orders` 리스트에서 자기 차례가 왔을 때, 이미 이전 순서의 대표자에게 공격당해 신앙심이 깎이고 음식이 바뀌었는데도, 무시하고 기존에 장전해둔 간절함으로 전파를 시도합니다.
- **핵심:** "방어 상태가 된 학생은 당일 전파를 하지 않는다"는 지문 규칙에 따라, 자신의 턴이 시작되자마자 **현재 기절(방어) 상태인지 체크하는 로직**이 가장 선행되어야 합니다.
- **➡️ 해결 방법:**
    - 전파 시 누군가를 공격(강/약 전파)했다면 대상의 좌표에 방어 도장(`v[nr][nc] = True`)을 찍습니다.
    - `for order in orders:` 반복문 시작 직후, `if v[r][c]: continue`를 배치하여 내가 방어 상태라면 간절함을 소모하지 않고 즉시 턴을 종료(스킵)합니다.
