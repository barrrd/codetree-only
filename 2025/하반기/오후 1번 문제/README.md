# [삼성 기출] AI 로봇청소기

> **문제 링크**: [코드트리 - AI 로봇청소기](https://www.codetree.ai/ko/frequent-problems/samsung-sw/problems/ai-robot/description)

---

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) Step 1: 이동 전 '현재 칸'의 오염 상태 우선 확인
로봇 청소기가 이동할 오염된 격자를 찾을 때, 탐색의 시작점인 **현재 자신의 위치**가 이미 오염되어 있는지 확인하는 것이 가장 중요합니다.

* **오류 예시:** 탐색을 시작하자마자 `carr[sr][sc] = 0`으로 현재 위치를 지우고, BFS의 인접 격자(`dist=1`)부터 탐색을 시작하는 경우입니다. 이 경우 현재 칸이 오염되어 있음에도 불구하고 다른 먼 칸으로 이동하는 논리적 모순이 발생합니다.
* **핵심:** '가장 가까운 오염된 격자'를 찾을 때 거리 0인 **현재 위치(`sr, sc`)**가 1순위 후보가 되어야 합니다.
* **➡️ 해결 방법:** * BFS 시작 전, `if arr[sr][sc] > 0:` 조건을 통해 현재 칸이 오염되었는지 먼저 확인합니다.
    * 현재 칸이 오염되었다면 이동 거리 0으로 간주하고 탐색을 생략한 채 그 자리에 머뭅니다(`er, ec = sr, sc`).
    * `carr` 업데이트 시에도 반드시 **[기존 위치 0 처리] -> [새 위치에 로봇 번호 기입]** 순서를 지켜 실시간 지도를 유지해야 합니다.

### 2) Step 1: BFS 내 '순차적 이동'과 장애물 판정
청소기들이 번호 순서대로 이동하며, 다른 청소기가 있는 칸은 지나갈 수 없다는 조건이 실시간으로 반영되어야 합니다.

* **오류 예시:** 모든 로봇의 이동 전 위치(`carr`)를 고정한 채 BFS를 수행하면, 이미 이동을 마친 앞 번호 로봇의 새 위치를 장애물로 인식하지 못해 **겹침 현상**이 발생합니다.
* **핵심:** 로봇 한 대가 이동을 마칠 때마다 `carr` 배열을 즉시 업데이트하여, 다음 로봇이 그 위치를 벽으로 인식하게 해야 합니다.
* **➡️ 해결 방법:**
    ```python
    for key in sorted(pos.keys()):
        sr, sc = pos[key]
        carr[sr][sc] = 0  # 1. 내 현재 위치 비우기
        # ... BFS 탐색 (실시간 carr 참조) 후 er, ec 결정 ...
        pos[key] = [er, ec]
        carr[er][ec] = key # 2. 내 새 위치 채우기 (다음 로봇이 참고)
    ```

### 3) Step 4: '합의 나눗셈' vs '나눗셈의 합' (확산 공식)
먼지 확산 시 연산 순서에 따라 결과값의 소수점 버림 처리가 달라져 큰 오차가 발생할 수 있습니다.

* **오류 예시:** `diff += arr[nr][nc] // 10` 처럼 각 주변 격자의 먼지를 각각 10으로 나눈 뒤 합산하는 방식입니다. (예: 주변 먼지가 9, 9일 때 `0 + 0 = 0`이 됨)
* **핵심:** 문제는 **'주변 4방향 격자의 먼지량 합을 10으로 나눈 값'**을 요구합니다. 즉, 인접 칸의 먼지를 모두 더한 후 마지막에 한 번만 정수 나눗셈(`// 10`)을 수행해야 합니다. (예: 주변 먼지가 9, 9일 때 `(9+9) // 10 = 1`이 됨)
* **➡️ 해결 방법:**
    ```python
    # 정답: (sum(주변 먼지)) // 10
    total_around = 0
    for dr, dc in [(-1,0), (1,0), (0,-1), (0,1)]:
        nr, nc = cr + dr, cc + dc
        if 0 <= nr < n and 0 <= nc < n:
            total_around += arr[nr][nc]
    
    arr[cr][cc] += (total_around // 10)
    ```

### 4) Step 2: 청소 방향 선택 우선순위 관리
먼지 합이 같은 방향이 여러 개일 때의 우선순위(오른쪽-아래쪽-왼쪽-위쪽)를 정확히 제어해야 합니다.

* **오류 예시:** 최댓값이 갱신될 때마다 방향을 바꾸거나(`if best <= tmp`), 탐색 순서를 무작위로 하는 경우입니다.
* **핵심:** 탐색 루프를 문제에서 제시한 우선순위 순서대로 돌리고, **최댓값이 처음 발견되었을 때만** 기록을 유지해야 합니다.
* **➡️ 해결 방법:**
    * 방향 벡터(`dr, dc`)를 `[오른쪽, 아래쪽, 왼쪽, 위쪽]` 순으로 정의합니다.
    * `for i in range(4):` 루프를 사용하고, `if best < tmp:` (등호 없음) 조건을 사용하여 먼저 발견된 높은 우선순위 방향이 유지되도록 합니다.
