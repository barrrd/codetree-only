# [삼성 기출] 택배 하차 

> > **문제 링크**: [코드트리 - 택배 하차 ](https://www.codetree.ai/ko/frequent-problems/samsung-sw/problems/delivery-service/description)

## ⚠️ 구현 시 주의해야 할 핵심 포인트

### 1) 객체 하차(삭제) 시의 정확한 경로 탐색 범위
객체를 특정 방향(좌/우)으로 추출할 때, 해당 객체가 나가는 경로에 다른 장애물이 없는지 확인하는 인덱스 범위 설정이 매우 중요합니다.

* **오류 예시 (우측 하차):** `for rc in range(sc, n):` 처럼 범위를 잡으면 자기 자신의 몸체까지 장애물로 인식하여 `False`를 반환하게 됩니다.
* **핵심:** 객체의 현재 점유 범위를 제외한 **'순수 경로'**만 체크해야 합니다.
* **➡️ 해결 방법:** * 왼쪽 하차: `range(0, sc)` (0번 열부터 객체 시작 열 전까지)
    * 오른쪽 하차: `range(sc + w, n)` (객체 끝 열 다음부터 격자 끝까지)

### 2) '공중 부양' 방지를 위한 전수 조사 및 연쇄 하강
객체가 하나 삭제되면, 그 객체에 의지하고 있던 위쪽 객체들뿐만 아니라 주변 객체들도 연쇄적으로 떨어질 수 있습니다.

* **오류 예시:** 삭제된 객체의 바로 윗부분(`range(0, sr)`)만 갱신하거나, 특정 범위만 `update` 할 경우 옆에 걸쳐 있던 객체가 허공에 떠 있는 논리적 오류가 발생합니다.
* **핵심:** 객체 하나가 하차될 때마다 **격자에 남은 모든 객체**를 다시 하강시켜야 합니다.
* **➡️ 해결 방법:**
    * 하차 성공 시, `pos.keys()` 전체를 후보로 잡습니다.
    * **중요:** 반드시 **가장 아래에 있는 객체(행 인덱스가 큰 것)부터** 순서대로 `update`를 수행해야 아래층부터 차곡차곡 쌓입니다. `cand_list.sort(key=lambda x: -x[0])`와 같은 정렬이 필수입니다.

### 3) `update` 시 자기 자신에 의한 충돌 판정 방지
이미 격자(`arr`)에 배치된 객체를 아래로 이동시킬 때, 자기 자신의 현재 위치 때문에 `can_down`이 `False`를 반환하는 현상을 방지해야 합니다.

* **오류 예시:** 기존 위치를 지우지 않고 `can_down(sr + 1, ...)`을 호출하면, 바로 아래 칸이 자기 자신의 몸체일 경우 장애물로 판단하여 이동하지 못합니다.
* **핵심:** **[기존 위치 지우기] -> [최하단 위치 찾기] -> [새 위치에 그리기]** 순서를 엄수해야 합니다.
* **➡️ 해결 방법:**
    ```python
    # 1. 현재 위치 초기화
    for r in range(cr, cr + h):
        for c in range(cc, cc + w):
            arr[r][c] = 0
    # 2. 이동 가능 여부 체크 및 sr 갱신
    while can_down(sr + 1, sc, h, w):
        sr += 1
    # 3. 최종 위치에 다시 그리기
    ```

### 4) 하차 우선순위 및 종료 조건 관리
문제 조건에 따른 우선순위(번호가 작은 순서 등)와 시뮬레이션의 반복 종료 시점을 정확히 제어해야 합니다.

* **오류 예시:** 하차할 객체가 여러 개일 때 `pos.items()`를 그대로 순회하면 입력 순서나 딕셔너리 내부 순서에 따라 결과가 달라집니다.
* **핵심:** 매 하차 시도마다 `sorted(pos.keys())`를 사용하여 문제에서 제시한 번호 우선순위를 보장해야 합니다.
* **➡️ 해결 방법:**
    * `while` 루프 내에서 한 사이클(좌측 하차 + 우측 하차) 동안 **단 한 건의 하차도 발생하지 않았다면** 더 이상 진행할 수 없는 상태로 간주하고 `break` 합니다.
    * 하차 함수(`left()`, `right()`)가 성공 여부(`True/False`)를 반환하게 하여 플래그 관리를 명확히 합니다.
